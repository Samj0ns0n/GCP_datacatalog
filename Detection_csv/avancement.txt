############ FEATURES ############

- Prend le premier fichier csv dans le bucket du cloud storage
	dans l'hypothèse que les fichiers d'un même bucket ai
	le même schéma

- Détecte les noms des colonnes, les types de colonnes d'un
	fichier csv en utilisant du pattern matching
	Detecte : Datetime/date/time/int/string/bool

- Supprime et recré les groupes d'entrées et les entrées avec
	le schéma automatiquement géré

- Supporte les csv avec ou sans header

########## BEMOL ####################

- Doit être actionnée à chaque fois qu'il a un changement dans
	le fichier csv/bucket/entré

- L'utilisateur doit choisir le nom du bucket/le nom du groupe
	d'entrée/le nom de l'entrée/l'id du groupe d'entree/
	/l'id de l'entré/la description du groupe d'entré 
	à chaque fois qu'il y a un changement

- Manque des credentials afin d'accéder aux données

- Lors de la création et suppression suppression des tags

- les patterns pour le pattern matching doit être crée en fonction
	des données ==> pour l'instant sur les données de lisea-mesea
	présent dans big query
	Nombres de pattern limités
